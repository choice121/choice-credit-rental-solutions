## Rules

1. **PNPM_VERSION must be `10`** — The lockfile is version `9.0` format (generated by pnpm 10). Setting `PNPM_VERSION=9` on Cloudflare Pages causes a silent build failure. Always keep this at `10` in CF Pages env vars (both preview and production).

2. **PORT must be optional in vite.config.ts** — CF Pages production builds don't pass a PORT env var. The vite config reads `const port = parseInt(process.env.PORT || "5000")` to avoid crashing.

3. **Build command:** `pnpm --filter @workspace/choice-credit run build`

4. **Output dir:** `artifacts/choice-credit/dist/public` — matches `build.outDir` in vite.config.ts.

5. **Required CF env vars:**
   - `PNPM_VERSION=10`
   - `NODE_VERSION=20`
   - `BASE_PATH=/`
   - `VITE_SUPABASE_URL=https://fahzqjhwhzdgswtfvjks.supabase.co`
   - `VITE_SUPABASE_ANON_KEY=<anon key>`

**Why:** CF Pages uses a separate build environment. It installs pnpm at the version specified in PNPM_VERSION before running the build. A version mismatch with the lockfile format causes an empty/silent failure with no log output available via the API.
